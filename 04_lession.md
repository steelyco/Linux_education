# Загрузчики Linux

При включении ПК инициализируется ПО материнской платы, которое после всех проверок передает управление первым секторам жесткого диска. В этих секторах находится файлы загрузчика - специальное ПО, позволяющее загрузить ядро ОС в оперативную память.

Раньше использовался MBR загрузчик - это был файл который запускал дальнейшие этапы загрузки ПК.

Сейчас используется GPT (GUID Partition Table) - таблица разделов на диске, которая использует глобально уникальные идентификаторы (GUID) для каждого раздела и всего диска.

Преимущества: 
- Есть резервная копия в конце диска
- Огромный вес 
- 128 разделов 
- Идентификация - GUID
- Информация о разделах

Внутри этой таблице разделов, помимо всего прочего, есть и - ESP (EFI System Partition) - это специальный раздел FAT32, который обязателен для загрузки UEFI+GPT. В этом разделе и лежит загрузчик операционной системы - GRUB2 (GRand Unified Bootloader version 2).

Файл конфигурации загрузчика лежит - /boot/grub/grub.cfg - Никак не редактируется 
Запускается с помощью скриптов - /etc/grub.d 
И настроек - /etc/default/grub
